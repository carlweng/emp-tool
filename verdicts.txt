# emp-tool design review — verdicts
# (multi-agent review, 2026-06-10: 9 deep-readers across emp-tool layers and
# consumers, 4 design critics, 52 merged claims each adversarially verified
# against the code: 44 confirmed, 5 refuted, 3 unverified.)

## Verdict

This is a good design — unusually good for a research library — and the review
tried hard to break it and mostly failed on the core. The three-layer
architecture is real and enforced, the 4-op BooleanContext concept is proven
right-sized by its two hardest consumers, and the claimed properties (static
dispatch, write-once-run-anywhere, session/value decoupling) all verified
against the code. The confirmed weaknesses cluster at the edges and the process
around the code, not the architecture: an unstated IR contract that ag2pc
re-derives by hand, a missing helper layer that all the sessions copy-paste,
documentation rot, and two consumers (emp-zk arith side, ref/emp-agmpc) whose
needs the current concepts don't yet express.

## The intention, reconstructed

The redesign replaces the old global-singleton model — CircuitExecution::circ_exec
/ ProtocolExecution::prot_exec virtual backends, block-hardwired Bit/Integer,
Bristol .txt files, a 49k-line embedded float table — with:

1. BooleanContext (ir/context/concept.h): four value-return ops over a regular
   Wire. No global, no virtual dispatch; heavy state lives in the context,
   keyed by a cheap handle.
2. Context-bound values Bit_T/UInt_T/Int_T/Float_T/BitVec_T<Ctx> carrying their
   own codec metadata, with hardware-matching numeric semantics.
3. One IR (BooleanProgram + validated .empbc), making user circuits exactly as
   portable as shipped builtins.
4. Pure-function frontend where I/O is structurally impossible inside a circuit
   body (RecordCtx has none).
5. Session layer owning the I/O boundary, with the reveal contract
   (std::optional<clear_t>, recipient-explicit) in the type system.

## How each consumer uses it (at review time)

- emp-ot      new layout, builds green. Pure runtime/ consumer (block,
              transpose, AES tiles, MITCCRH, f2k, NetIO) via the top umbrella.
- emp-sh2pc   fully migrated. The design's proof point: a complete GC backend
              in 368 lines — SH2PCCtx is a 77-line BooleanContext calling
              halfgates_garble/eval; the session owns Delta/OT/MITCCRH and the
              input/reveal boundary. Old-world equivalent (ref): ~4,771 lines.
- emp-ag2pc   fully migrated, deepest IR consumer. WRK17 runs five protocol
              passes (liveness, slot-mask, garble, evaluate, gamma-check), each
              itself a BooleanContext, all through the generic execute_program —
              it never iterates gates itself. The hardest consumer; the seam
              held with zero IR edits.
- emp-zk      bool side natively migrated (f896deb: ZKBoolContext + ZKBoolSession).
              Arithmetic side still pre-redesign (ZKFpExec global singleton).
- ref/agmpc   old layout, broken: three subclasses of the deleted emp::Backend +
              BackendGuard global swap. Hand-rolls everything multiparty
              (NetIOMP mesh, echo-broadcast, n-party shares); nothing upstream
              represents n parties yet.

## Verified strengths

- The layering is checkable, not aspirational: ir/ touches runtime/ only
  through error() and constants; two zero-include design roots (concept.h,
  program.h) anchor the seams.
- The concept is right-sized: small enough for a 78-line backend, expressive
  enough for a five-pass malicious protocol. Context / Session / WireValue /
  circuit_fn each own a distinct axis.
- Static dispatch is real: the per-gate path from typed value to AES tile has
  exactly one virtual call (IOChannel, amortized by 32 KiB staging). Measured
  compile cost of the template design: ~1.3-1.8 s per TU at -O2.
- The refactor-safety methodology is right for crypto: identical gate streams
  across contexts + digest_source == digest_program determinism oracle +
  TraceIO wire-byte diffing catch transcript-order bugs output tests can't.
- Rare guardrails: compile-fail probe tests pin the contract diagnostics;
  .empbc is versioned with one hardened loader; kernels are concrete-security-
  literate (ReuseShift's 3-bit loss quantified in-header).
- Recipient-explicit optional reveal kills the old decoded-placeholder leak.

Five critic claims died under adversarial verification — itself evidence:
the "1.0.0 versioning is inoperative" framing (v0.3.x freeze + pre-alpha dev
branch is documented), "B3 is blocked on unsolved upstream design" (extension
points exist), and "float comparisons garble 7 dead output bits" (measured:
the dead cones contain only free XOR/NOT — zero wasted ANDs in all 30
comparison assets).

## Confirmed issues (prioritized)

1. Name the IR's real contract. validate_program guarantees dense/topological/
   single-def, but the stronger record-canonical shape (gate.out ==
   num_inputs+i, deduped consts) that ag2pc's multi-pass design depends on is
   documented nowhere upstream — ag2pc hand-rolls ag2pc_require_record_canonical
   and rejects valid programs rather than adapting. Ship validate_canonical()
   + canonicalize() + a shared recorder core (the id-counter/const-dedup idiom
   is hand-mirrored across RecordCtx, DigestCtx, and six ag2pc copies).

2. [Maintainer: deferred — API intentionally unstable pre-alpha, no users
   expected until things are fixed.] Versioning/stability discipline.

3. Design B3/B4's missing pieces upstream-first: ZK FieldContext (ops =
   ZKFpExec's gate virtuals: add/mul/mul_const/public_elem; the feed/reveal_check
   family is session I/O; Δ-sharing is per-compatible-field), and n-party
   readiness (the session concepts are already party-count-agnostic; allow
   n-party ids in the base session contract; upstream NetIOMP with the port).

4. Specify reveal's malicious-security semantics: a revealed value is
   provisional until the session's documented settlement point (AG2PC settles
   at reveal; ZK at finalize()); error() is the malicious-abort semantic.

5. Ship the missing session helper layer (small free functions only): the
   typed-I/O codec skeleton is copied in ClearSession/SH2PC/AG2PC/ZK sessions;
   std::regular Wire should drop to semiregular — wire equality is used
   nowhere and it alone forces sh2pc's SHWire wrapper.

6. Make the safety claims mechanically true: visit.h's "only op switch" claim
   is false (op dispatch in 6 places; digest_program's ternary silently hashes
   unknown ops as Const0); add -Werror=switch. Unify the failure model
   (uniform error(), no exceptions — maintainer decision).

7. Docs rot: AGENTS.md partially stale; backend.md says ctx() where the
   concept requires direct_ctx(); EMP_TRANSLATION.md denies shifts/resize that
   unsigned_int.h now ships; passes.h provenance comments. The 63 committed
   .empbc binaries have no in-tree regeneration pipeline [deferred]; asset
   lookup bakes configure-time absolute paths [embed builtins — lower priority].

8. What emp-tool should own (convergent duplication): the Mersenne-2^61-1
   field kit (emp-ot/svole/fp_utility.h, consumed by emp-zk-arith), one
   exported Δ-LSB/point-and-permute constant (re-derived at 5 sites), an
   AuthValue val-low/mac-high layout trait replacing the comment-only
   __uint128_t ABI between emp-ot and emp-zk.

Inversion worth noting: the speculative surface (BulkBooleanContext/
ScheduledPlan for an unwritten GMW backend, consumer-less liveness/layout
passes) has zero consumers, while surface consumers converged on
(canonicalize, batch I/O, wide codecs, F_p) didn't exist upstream.

## Bottom line

The architecture has earned the migration — sh2pc and ag2pc prove it on both
the easy and hard ends. What stands between "impressive working tree" and
"substrate that carries the zk-arith and agmpc ports" is edge work: name the
canonical contract, ship the minimal session helpers, write the FieldContext/
n-party design docs first, and stop the docs rot.
