emp-tool implementation audit

A high-level overview plus three implementation-level reports for an external auditor — one per architectural layer. emp-tool is organized as three layers with a strict dependency order circuits → ir → runtime: the runtime substrate (SIMD block, AES / PRG / PRP / hash / GF(2¹²⁸) / EC kernels, IO channels, garbling leaf primitives), the context-free Boolean IR (programs, contexts, the WireValue and session contracts), and the concrete circuit value families plus the compile-once / run-on-any-context frontend. There is no global backend.

Report 1 · runtime

Runtime substrate

The SIMD block type & helpers; AES-128 key schedule / ECB, the fixed-key PRP, AES-CTR PRG, CCRH / MITCCRH, GF(2¹²⁸) f2k, SHA-256 & P-256 (OpenSSL), the random oracle; the NetIO / TLSIO / TraceIO channels; and the half-gate / privacy-free garbling leaf primitives.
Report 2 · ir

Boolean IR, contexts & sessions

The C++20 BooleanContext concept and the reusable contexts (Clear / Count / Digest / Record), the flat .empbc IR (program / validate / visit / passes / schedule / execute / artifact / assets), the generic WireValue concept, and the Session / DirectSession / SessionIO / CheckpointingSession contracts + ClearSession.

Circuit value objects & frontend
